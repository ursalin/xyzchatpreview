# 第一课补充：常见概念答疑

## Q1: src 是什么？

`src` = **source**（源代码）的缩写，就是"放代码的地方"。

### 项目完整结构
```
xyzchatpreview/
├── src/           ← 你写的代码都在这里 ★
├── public/        ← 静态资源（图片、图标等）
├── node_modules/  ← 第三方库（自动下载，不用管）
├── package.json   ← 项目说明书（依赖列表、启动命令）
└── vite.config.ts ← 构建工具配置
```

**重点：你只需要关心 `src/` 就行，其他都是"基础设施"。**

## Q2: 为什么逻辑模块叫 Hook（钩子）？

Hook = 让组件**钩住**某种能力的工具。

### 理解方式
组件（积木）本身只管显示 UI，但它需要"钩"一些能力进来：
- `useChat` — 钩住聊天能力（发消息、收回复）
- `useVideoCall` — 钩住视频通话能力（摄像头、录音、TTS）
- `useSettings` — 钩住设置数据（角色名、API 配置）

### React 规定
所有 Hook 必须以 `use` 开头，看到 `use` 就知道"这是一个可以被组件钩进来用的逻辑模块" 🪝

## Q3: Node.js 到底是啥？

JavaScript 原本只能在浏览器里跑，Node.js 让 JS 可以在电脑/服务器上直接运行。

### 对比
| 环境 | 作用 |
|------|------|
| **浏览器里的 JS** | 控制网页（点按钮、显示内容） |
| **Node.js 里的 JS** | 控制电脑（读文件、装工具包、跑服务器） |

### 对咱们项目的作用
1. **npm/包管理** — `npm install` 下载第三方库
2. **开发工具** — Vite（打包工具）跑在 Node.js 上

**你不需要写 Node.js 代码，但它是开发环境的"地基"。** 就像你不需要会修水管，但家里得有自来水 🚰

## Q4: npm 是什么缩写？

**npm** = **N**ode **P**ackage **M**anager（Node 包管理器）

### 概念
- **包（package）** = 别人写好的代码工具，你可以直接拿来用
- **npm** = 命令行里的 App Store
- **node_modules/** = 已安装的包存放位置
- **package.json** = "购物清单"，记录项目需要哪些包

### 类比
| 手机 | 前端项目 |
|------|----------|
| App Store | npm |
| 一个个 App | 一个个包（react、axios 等） |
| 已安装的 App | node_modules/ 文件夹 |

### 常用命令
```bash
npm install react        # 安装 react 包
npm install              # 安装 package.json 里列出的所有包
npm run dev              # 启动开发服务器
```

## Q5: npm 相当于命令行里的 App Store 吗？

**对！** npm 就是命令行里的包管理器。

### 各系统的包管理器对比
| 系统/场景 | 包管理器 | 安装命令 |
|-----------|----------|----------|
| macOS | Homebrew | `brew install xxx` |
| Ubuntu/Linux | APT | `apt install xxx` |
| Windows | Chocolatey / winget | `choco install xxx` |
| **前端项目** | **npm** | `npm install xxx` |

## Q6: PowerShell 到底算个啥？

PowerShell 是 Windows 的**命令行工具**——用文字命令控制电脑。

### 图形界面 vs 命令行
| 类型 | 比喻 | 特点 |
|------|------|------|
| **图形界面（GUI）** | 精装修的房子 | 直观、友好，鼠标点点点 |
| **命令行（CLI）** | 毛坯房 | 灵活、快速、可自动化 |

### 各系统的命令行
- **Windows** → PowerShell 或 CMD
- **macOS/Linux** → Terminal（终端），用 Bash/Zsh

### 为什么开发要用命令行？
很多开发工具（npm、git、vercel）都是命令行工具，没有图形界面。

**例子：**
```bash
npm install              # 安装依赖包
git push                 # 推代码到 GitHub
vercel deploy            # 部署网站
```

### 命令行的优势
- **灵活** — 想干啥干啥，不受限制
- **快** — 不用渲染界面，直接干活
- **自动化** — 可以写脚本批量执行（比如一次性部署100个网站）

**PowerShell = Windows 的"毛坯操作台"，给你最底层的控制权** 💪

## 关键词速查（补充）

- **src/**: 源代码文件夹
- **Hook**: React 的逻辑复用机制，以 `use` 开头
- **Node.js**: 让 JavaScript 可以在电脑上运行的环境
- **npm**: Node 包管理器，命令行里的 App Store
- **package.json**: 项目依赖清单
- **node_modules/**: 已安装的第三方包存放位置
- **PowerShell**: Windows 的命令行工具
- **CLI**: Command Line Interface（命令行界面）
- **GUI**: Graphical User Interface（图形用户界面）
